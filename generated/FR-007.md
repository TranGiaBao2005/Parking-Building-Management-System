# FR-007.md - Parking Session Tracking and Fee Calculation

## SECTION 1: FUNCTIONAL & BUSINESS LOGIC ANALYSIS

### CRUD Matrix

| Operation | Entity | Conditions | Business Rules | Audit |
|-----------|--------|-----------|-----------------|-------|
| **READ** | parking_session | • Session exists<br>• Driver owns or Manager | • Return all details<br>• Calculate live duration<br>• Calculate live fee estimate | Log: Session view, user_id |

### Data Dictionary (See FR-005 for parking_sessions base schema)

### Business Rules

1. **Live Fee Estimation**
   - Real-time calculation: current_duration = now() - entry_time
   - Formula: ceil(current_duration / 3600) × hourly_rate
   - Update every 30 seconds via WebSocket or polling
   - Show estimate ±1% accuracy
   - Include surcharges if applicable (overdue, etc.)

2. **Session History**
   - Retain past sessions: minimum 24 months
   - Driver can filter by date range
   - Show all sessions for own plate number

### RBAC

| Role | Own Sessions | All Sessions | Export History |

| **Manager** | N/A | ✅ Facility | ✅ |
| **Staff** | N/A | ✅ Facility | ✅ |
| **Driver** | ✅ | ❌ | ✅ Own |

## SECTION 3: BACK-END SPECIFICATIONS

### Database Schema

```sql
CREATE TABLE session_tracking_events (
    event_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    session_id UUID NOT NULL REFERENCES parking_sessions(session_id),
    event_type VARCHAR(50) NOT NULL,
    event_timestamp TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    details_json JSONB,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC'
);

CREATE INDEX idx_tracking_session ON session_tracking_events(session_id);
CREATE INDEX idx_tracking_event_type ON session_tracking_events(event_type);
```

## SECTION 4: ACCEPTANCE CRITERIA

### Scenario 1: View Current Session (Happy Path)
```gherkin
Given active parking session
When driver navigates to SessionTrackingPage
Then system displays:
  - Vehicle plate and type
  - Entry time (formatted for timezone)
  - Live duration (updates every 30 sec)
  - Assigned slot
  - Live fee estimate
  - Entry time elapsed timer
And estimate updates every 30 seconds
And response time ≤1 second
```

### Scenario 2: Live Fee Estimate Updates (Happy Path)
```gherkin
Given live fee estimate = 100000
After 1 hour (3600 seconds)
When fee estimate recalculates
Then estimate = 200000 (2 hours)
And update is displayed within 30 seconds
And shows calculation breakdown
```

### Scenario 3: View Session History (Happy Path)
```gherkin
Given driver with 10 past parking sessions
When driver views SessionHistoryPage with date filter
Then system displays all sessions with:
  - Entry time, exit time, duration, fee
  - Date range filtering
  - Sorting by date
  - Pagination if >20 sessions
```

### Scenario 4: Authorization Check (Edge Case)
```gherkin
Given Driver A viewing Driver B's sessions
When Driver A attempts GET /api/v1/drivers/sessions/other_plate
Then system returns HTTP 403 Forbidden
And displays error: "Cannot access other driver's sessions"
And no session data returned
```

### Scenario 5: Fee Estimate with Overdue Surcharge (Edge Case)
```gherkin
Given parking session >24 hours with overdue policy
When fee estimate calculated
Then displays:
  - Base fee
  - Overdue surcharge (hourly rate × hours_over_limit)
  - Total estimate with surcharge
```
