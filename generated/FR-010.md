# FR-010.md - Reports and Analytics

## SECTION 1: FUNCTIONAL & BUSINESS LOGIC ANALYSIS

### CRUD Matrix

| Operation | Entity | Conditions | Business Rules | Audit |
|-----------|--------|-----------|-----------------|-------|
| **READ** | analytics_data | • User: Manager+<br>• Own facility (Manager) | • Aggregate parking sessions and payments<br>• Calculate occupancy, revenue, volume<br>• Group by hour/day/week<br>• Identify peak hours | Log: Report generated, user_id |

### Report Types

1. **Entry/Exit Volume Report**
   - Count entries and exits per hour/day
   - Group by vehicle type, gate
   - Show trends over time

2. **Revenue Report**
   - Sum of all completed payments
   - Breakdown by payment method (CASH, CARD, etc.)
   - Breakdown by vehicle type
   - Breakdown by time period

3. **Occupancy Report**
   - Occupancy rate over time (hourly, daily)
   - Peak hours identification (≥80% of daily average)
   - Occupancy by floor/section
   - Trend analysis

4. **Exceptions Report**
   - Count by exception type: LOST_TICKET, WRONG_PLATE, OVERDUE, WRONG_ZONE, UNPAID
   - Timeline of exceptions
   - Pattern analysis

### Business Rules

1. **Data Aggregation**
   - Denormalize into analytics_aggregates table (nightly job)
   - Store metrics: ENTRY_COUNT, EXIT_COUNT, OCCUPANCY_RATE, REVENUE, AVG_DURATION
   - Grouping: hourly (≤7 days), daily (≤31 days), weekly (>31 days)

2. **Peak Hours Definition**
   - Peak: occupancy ≥ 80% of daily average occupancy for that vehicle type
   - Example: if daily average = 60%, then peak = ≥48% occupied

3. **Timezone Handling**
   - Store all times in UTC
   - Display in facility timezone
   - Reports use facility timezone for grouping

4. **Export Formats**
   - CSV: standard comma-separated, UTF-8 encoding
   - PDF: tabular format with charts (Phase 2)
   - JSON: for programmatic access

## SECTION 3: BACK-END SPECIFICATIONS

### Database Schema

```sql
CREATE TABLE analytics_aggregates (
    aggregate_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    facility_id UUID NOT NULL REFERENCES facilities(facility_id),
    report_date DATE NOT NULL,
    report_hour INTEGER, -- NULL for daily, 0-23 for hourly
    report_week INTEGER, -- ISO week number
    metric_type VARCHAR(50) NOT NULL,
    metric_value DECIMAL(12, 2),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC'
);

-- Metric types: ENTRY_COUNT, EXIT_COUNT, OCCUPANCY_RATE, REVENUE, AVG_DURATION

CREATE INDEX idx_aggregates_facility_date ON analytics_aggregates(facility_id, report_date);
CREATE INDEX idx_aggregates_metric ON analytics_aggregates(metric_type);
```

## SECTION 4: ACCEPTANCE CRITERIA

### Scenario 1: Generate Revenue Report (Happy Path)
```gherkin
Given manager requests revenue report for Jan 1-31
When report generated with date range
Then system displays:
  - Total revenue: 125.45M VND
  - Average daily: 4.05M VND
  - Breakdown by payment method (%)
  - Top vehicle type by revenue
  - Response time ≤5 seconds
And returns HTTP 200
```

### Scenario 2: Identify Peak Hours (Happy Path)
```gherkin
Given occupancy data for full month
When system analyzes peak hours
Then identifies hours where occupancy ≥ 80% of daily average
And displays peak hours (e.g., 12:00-15:00)
And shows average occupancy during peak
And allows filtering by vehicle type
```

### Scenario 3: Large Date Range (Edge Case)
```gherkin
Given date range 365 days (entire year)
When manager requests report
Then system groups data by WEEK (not hourly)
And response time ≤5 seconds
And displays weekly aggregated metrics
And prevents timeout on large datasets
```

### Scenario 4: Export Report to CSV (Happy Path)
```gherkin
Given revenue report displayed
When manager clicks [Download CSV]
Then system generates CSV file
And returns Content-Type: text/csv
And filename: "revenue_report_20260101_20260131.csv"
And can be opened in Excel
```

### Scenario 5: Authorization Check (Edge Case)
```gherkin
Given driver attempting to view manager reports
When driver requests GET /api/v1/reports/revenue
Then system returns HTTP 403 Forbidden
And displays error: "Insufficient permissions"
And no data returned
```

---

## PERFORMANCE REQUIREMENTS

- Report generation: ≤5 seconds (date range ≤90 days)
- CSV export: <10MB file size, ≤10 seconds generation
- Aggregation job: nightly run <1 hour for all facilities
- Query optimization: use aggregates table, not raw sessions
- Caching: cache hourly reports for 1 hour

---

## DATA RETENTION & ARCHIVAL

- Live data: 3 months (rolling)
- Aggregated metrics: 36 months minimum
- Archive old data quarterly to cold storage
- Maintain audit trail for historical analysis
