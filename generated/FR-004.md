### Business Rules & Constraints

1. **Rate Hierarchy**
   - hourly_rate must be base; daily = hourly × 8-12 (manager choice); monthly = daily × 20-30
   - Grace period: 0-30 minutes (typical 0-5 for parking)
   - Validation: hourly rate ≥ 10,000 VND; ≤ 500,000 VND

2. **Time Period Logic**
   - Peak: defined by manager-configurable time windows (e.g., 08:00-10:00, 17:00-19:00)
   - Off-peak: outside peak hours
   - Weekend: Saturday-Sunday
   - Weekday: Monday-Friday
   - Multiple policies can overlap; use priority: PEAK > WEEKDAY/WEEKEND > OFF_PEAK

3. **Fee Calculation**
   - Formula: ceil(duration_minutes / 60) × hourly_rate (or daily if ≥24hrs)
   - Rounding: to nearest 1000 VND (half-up)
   - Grace period: if duration ≤ grace_period_minutes, charge 0; else full fee
   - Fixed-point arithmetic (not floating-point) to avoid precision errors

## SECTION 3: BACK-END SPECIFICATIONS

### Database Schema

```sql
CREATE TABLE pricing_policies (
    policy_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    facility_id UUID NOT NULL REFERENCES facilities(facility_id),
    vehicle_type_id UUID NOT NULL REFERENCES vehicle_types(type_id),
    time_period VARCHAR(20) NOT NULL CHECK (time_period IN ('PEAK', 'OFF_PEAK', 'WEEKEND', 'WEEKDAY')),
    rate_per_hour DECIMAL(8, 2) NOT NULL CHECK (rate_per_hour BETWEEN 10000 AND 500000),
    daily_rate DECIMAL(8, 2),
    monthly_rate DECIMAL(8, 2),
    grace_period_minutes INTEGER NOT NULL DEFAULT 0 CHECK (grace_period_minutes BETWEEN 0 AND 30),
    active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    UNIQUE(facility_id, vehicle_type_id, time_period),
    CONSTRAINT daily_rate_valid CHECK (daily_rate IS NULL OR daily_rate >= rate_per_hour * 8),
    CONSTRAINT monthly_rate_valid CHECK (monthly_rate IS NULL OR monthly_rate >= daily_rate * 20)
);

CREATE TABLE fee_surcharges (
    surcharge_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    facility_id UUID NOT NULL REFERENCES facilities(facility_id),
    surcharge_type VARCHAR(50) NOT NULL,
    amount DECIMAL(8, 2) NOT NULL,
    description VARCHAR(200),
    active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC'
);

CREATE INDEX idx_pricing_facility ON pricing_policies(facility_id);
CREATE INDEX idx_pricing_vehicle_type ON pricing_policies(vehicle_type_id);
CREATE INDEX idx_surcharges_facility ON fee_surcharges(facility_id);
```

## SECTION 4: ACCEPTANCE CRITERIA

### Scenario 1: Calculate Fee with Grace Period (Happy Path)
```gherkin
Given policy with hourly_rate=100000, grace_period_minutes=5
When driver parks for 4 minutes
Then fee = 0 VND (within grace period)
When driver parks for 65 minutes
Then fee = 100000 VND (exceeds grace, charges 1 full hour)
```

### Scenario 2: Calculate Daily Rate (Happy Path)
```gherkin
Given policy with hourly_rate=100000, daily_rate=700000
When driver parks for 24 hours
Then fee = 700000 VND (not 2400000)
```

### Scenario 3: Calculate with Rounding (Edge Case)
```gherkin
Given hourly_rate=100000
When driver parks for 37 minutes
Then fee = ceil(37/60) × 100000 = 100000 VND (rounded up)
When driver parks for 1 hour 5 minutes
Then fee = ceil(65/60) × 100000 = 200000 VND (rounds to 2 full hours)
```

### Scenario 4: Prevent Invalid Policy (Edge Case)
```gherkin
Given Manager attempting to create policy with:
  | rate_per_hour | daily_rate |
  | 100000 | 500000 |
Then system rejects (500000 < 100000 × 8 = 800000)
And returns HTTP 400
And error message: "Daily rate must be ≥ hourly rate × 8"
```

### Scenario 5: Apply Surcharges (Happy Path)
```gherkin
Given base_fee = 100000 VND
When exit fee calculation includes WRONG_ZONE_PENALTY (20%)
Then total_fee = 100000 + (100000 × 0.20) = 120000 VND
And breakdown shows surcharge detail
```
