# FR-008.md - View Parking Facility Information (User)

## SECTION 1: FUNCTIONAL & BUSINESS LOGIC ANALYSIS

### CRUD Matrix

| Operation | Entity | Conditions | Business Rules | Audit |
|-----------|--------|-----------|-----------------|-------|
| **READ** | facility_info_public | • No authentication | • Return cached data<br>• Max 5-minute staleness<br>• Aggregate availability<br>• No sensitive data | Log: Public info view, facility_id |

### Business Rules

1. **Public Data Exposure**
   - Name, address, phone, email (no manager names)
   - Operating hours
   - Vehicle types served with pricing
   - Current availability summary (not detailed slot-by-slot)
   - Occupancy percentage (0-100%)

2. **Caching Strategy**
   - Availability cache: refresh on every entry/exit OR every 5 minutes
   - Response time: ≤500ms (use cache)
   - Invalidate cache when: slot status changes, pricing updated

3. **Performance**
   - High volume expected (drivers checking availability)
   - Use denormalized facility_info_cache table
   - Minimal database queries

## SECTION 3: BACK-END SPECIFICATIONS

### Database Schema

```sql
CREATE TABLE facility_info_cache (
    cache_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    facility_id UUID NOT NULL UNIQUE REFERENCES facilities(facility_id),
    name VARCHAR(255) NOT NULL,
    address VARCHAR(500),
    phone VARCHAR(15),
    email VARCHAR(255),
    total_capacity INTEGER,
    occupied_slots INTEGER,
    occupancy_percentage DECIMAL(5, 2),
    availability_json JSONB,
    pricing_json JSONB,
    operating_hours_json JSONB,
    last_updated TIMESTAMP NOT NULL DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC'
);

CREATE INDEX idx_cache_facility ON facility_info_cache(facility_id);
CREATE INDEX idx_cache_updated ON facility_info_cache(last_updated);
```

## SECTION 4: ACCEPTANCE CRITERIA

### Scenario 1: View Public Facility Info (Happy Path)
```gherkin
Given driver navigates to facility info page
When system loads facility details
Then displays:
  - Facility name, address, phone, email
  - Operating hours
  - Total capacity and occupancy
  - Occupancy rate (0-100%)
  - Vehicle types served with pricing
And returns HTTP 200 within 500ms
And no authentication required
And data freshness ≤5 minutes
```

### Scenario 2: View Current Availability (Happy Path)
```gherkin
Given facility page loaded
When driver views availability section
Then displays:
  - Occupancy progress bar
  - Percentage occupied
  - Available slots by vehicle type
  - "Full" indicator if all occupied
  - Last updated timestamp
And responds within 500ms
```

### Scenario 3: Check Pricing (Happy Path)
```gherkin
Given driver viewing pricing section
When pricing table displayed
Then shows:
  - Vehicle types
  - Hourly rates (VND, formatted)
  - Daily rates (if available)
  - Monthly rates (if available)
  - Grace period (if applicable)
And pricing matches current facility policies
```

### Scenario 4: Cache Invalidation (Edge Case)
```gherkin
Given cached facility info (3 minutes old)
When new vehicle exits (slot becomes VACANT)
Then system refreshes availability cache
And next public query shows updated occupancy
And cache age ≤ 5 minutes
```

### Scenario 5: Facility Not Found (Edge Case)
```gherkin
Given facility_id that doesn't exist
When driver requests /api/v1/public/facilities/invalid-id
Then system returns HTTP 404
And displays error: "Facility not found"
```
