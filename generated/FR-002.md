### Business Rules & Constraints

1. **Vehicle Type Management**
   - Type names must be unique per facility (not globally)
   - Size category must be one of: SMALL, MEDIUM, LARGE, EXTRA_LARGE
   - Dimensions (height, width, length) represent maximum allowed for vehicle type:
     - SMALL: typical motorcycle, 2-wheeler
     - MEDIUM: typical sedan car
     - LARGE: SUV, van
     - EXTRA_LARGE: truck, commercial vehicle
   - Base hourly rate must be positive VND (minimum 10,000; maximum 500,000)
   - Cannot delete vehicle type if active sessions using it (soft delete only)

2. **Floor/Section Allocation**
   - Floor numbers: 1-50 (typical building heights)
   - Sections must be sequential (can't skip: 1, 2, 3... no 1, 2, 5)
   - Section codes must follow format: [LETTER]-[NUMBER] (e.g., A-01, B-02)
   - Total slots per section: 1-999 (reasonable per section)
   - Sections must not exceed facility max_capacity when summed:
     - SUM(all_sections.total_slots) ≤ facility.max_capacity
   - Each section assigned to exactly one vehicle type (no mixed types per section)

3. **Capacity Management**
   - Validation on allocation: prevent over-allocation
   - Warning: section > 200 slots (unusually large)
   - Warning: section < 5 slots (unusually small)
   - When reducing section slots, prevent going below current occupancy

### RBAC (Role-Based Access Control)

| Role | Vehicle Type | Section Allocation | Notes |
|------|---|---|---|
| **Admin** | ✅ C/R/U/D | ✅ C/R/U/D | Full control, can delete |
| **Manager** | ✅ C/R/U | ✅ C/R/U | Can create/update, cannot delete |
| **Staff** | ❌ | ❌ | Read-only for informational purposes |
| **Driver** | ❌ | ❌ | No access |

**Error Handling**
- **400 Bad Request**: Invalid dimensions, rates, floor numbers
- **403 Forbidden**: Insufficient role
- **404 Not Found**: Facility, type, or section not found
- **409 Conflict**: Name already exists, over-allocation, invalid transitions

## SECTION 3: BACK-END SPECIFICATIONS

### Database Schema Design

```sql
CREATE TABLE vehicle_types (
    type_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    facility_id UUID NOT NULL REFERENCES facilities(facility_id),
    type_name VARCHAR(100) NOT NULL,
    size_category VARCHAR(20) NOT NULL CHECK (size_category IN ('SMALL', 'MEDIUM', 'LARGE', 'EXTRA_LARGE')),
    height_limit_cm INTEGER NOT NULL CHECK (height_limit_cm BETWEEN 0 AND 999),
    width_limit_cm INTEGER NOT NULL CHECK (width_limit_cm BETWEEN 0 AND 999),
    length_limit_cm INTEGER NOT NULL CHECK (length_limit_cm BETWEEN 0 AND 999),
    base_rate_per_hour DECIMAL(8, 2) NOT NULL CHECK (base_rate_per_hour >= 0),
    active BOOLEAN DEFAULT true,
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    UNIQUE(facility_id, type_name),
    CONSTRAINT valid_dimensions CHECK (height_limit_cm > 0 AND width_limit_cm > 0 AND length_limit_cm > 0)
);

CREATE TABLE facility_sections (
    section_id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    facility_id UUID NOT NULL REFERENCES facilities(facility_id),
    section_name VARCHAR(100) NOT NULL,
    floor_number INTEGER NOT NULL CHECK (floor_number BETWEEN 1 AND 50),
    section_code VARCHAR(20) NOT NULL,
    vehicle_type_id UUID NOT NULL REFERENCES vehicle_types(type_id),
    total_slots INTEGER NOT NULL CHECK (total_slots BETWEEN 1 AND 9999),
    created_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    updated_at TIMESTAMP DEFAULT CURRENT_TIMESTAMP AT TIME ZONE 'UTC',
    UNIQUE(facility_id, section_code),
    UNIQUE(facility_id, floor_number, section_code)
);

CREATE INDEX idx_vehicle_types_facility ON vehicle_types(facility_id);
CREATE INDEX idx_sections_facility ON facility_sections(facility_id);
CREATE INDEX idx_sections_vehicle_type ON facility_sections(vehicle_type_id);
```

### Exception Handling & HTTP Status Codes

| Scenario | HTTP Status | Response |
|----------|------------|----------|
| Duplicate vehicle type name per facility | 409 | `{"error": "Vehicle type name already exists"}` |
| Over-allocation (SUM slots > facility capacity) | 409 | `{"error": "Allocation exceeds facility capacity by X slots"}` |
| Invalid size_category | 400 | `{"error": "Invalid size category"}` |

| Vehicle type not found | 404 | `{"error": "Vehicle type not found"}` |
| Duplicate section code per facility | 409 | `{"error": "Section code already exists for this facility"}` |

## SECTION 4: ACCEPTANCE CRITERIA (BDD Format)

### Scenario 1: Create Vehicle Type (Happy Path)
```gherkin
Given a Manager authenticated for Facility A
When the Manager creates a vehicle type:
  | type_name | size_category | height | width | length | rate |
  | Ô tô 4 chỗ | MEDIUM | 180 | 200 | 500 | 100000 |
Then the system creates vehicle type record
And returns HTTP 201 with type_id
And displays "Vehicle type created successfully"
And vehicle type appears in the list
```

### Scenario 2: Allocate Section to Vehicle Type (Happy Path)
```gherkin
Given vehicle type "Xe máy" exists in Facility A
When the Manager allocates:
  | floor_number | section_code | section_name | total_slots |
  | 1 | A-01 | Tầng 1 - Xe máy | 150 |
Then the system creates section with allocation
And SUM(sections) remains ≤ facility.max_capacity
And section appears in allocation matrix
And returns HTTP 201
```

### Scenario 3: Prevent Over-Allocation (Edge Case)
```gherkin
Given Facility A with max_capacity 500
And existing allocations total 450 slots
When Manager attempts to add section with 75 slots
Then the system rejects allocation
And returns HTTP 409
And displays error: "Allocation exceeds facility capacity by 25 slots"
And no section record is created
```

### Scenario 4: Prevent Duplicate Vehicle Type Name (Edge Case)
```gherkin
Given vehicle type "Xe máy" exists
When Manager attempts to create another "Xe máy"
Then the system rejects creation
And returns HTTP 409
And displays error: "Vehicle type name already exists"
And only one record exists
```

### Scenario 5: Validate Section Code Format (Edge Case)
```gherkin
Given a Manager creating a section
When the Manager enters invalid section_code: "A1" (missing hyphen)
Then the system shows validation error: "Format: A-01"
And form submission is disabled
When the Manager corrects to "A-01"
Then validation error disappears
And form submission is enabled
```

### Scenario 6: Update Vehicle Type Rate (Happy Path)
```gherkin
Given vehicle type "Ô tô 4 chỗ" with rate 100000
When Manager updates rate to 120000
Then the system persists the change
And returns HTTP 200
And new sessions use the updated rate
And historical rates are preserved for reporting
```
